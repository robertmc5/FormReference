HTML Form Reference Guide

Form Tags
Tag Element Description
<form> Form It represents a document section containing interactive controls for submitting information.
<input> Input It is used to create interactive controls for web-based forms in order to accept data from the user.
<label> Input Label It provides identification for a specific <input> based on matching values of the <input>‘s [id] attribute and the <label>‘s [for] attribute.
<select> Select It is used to create a dropdown list. {<option>s are nested within it}
<option> Option It is used to define an item contained in a <select> or a <datalist> element, and is nested within it. {Data List <option>s have no content}
<datalist> Data List It contains a set of <option> elements that represent the permissible or recommended options available to choose from within other controls.
<textarea> Text Area It represents a multi-line plain-text editing control, useful when you want to allow users to enter a sizeable amount of free-form text.

Form Attributes
In Element Tag Attribute Name Description 
<form   >
 action=" " Action It determines where the <form> information is sent. ie. the path to the server
method=" " Method It is assigned a HTTP verb that is included in the HTTP request. eg. GET or POST 
<input   >
 type=" " Type It determines how it renders on the web page and what kind of data the <input> can accept. eg. text or number
name=" " Name A name is required for a submittal of the <input> field. ie. text with "name=value" is sent with [value] being the user input
id=" " ID It identifies the input, so that it's paired label's for=" " attribute can pair to the <input> or <select> with that ID.
value=" " Value It can assign a default value as a pre-filled text field or an assigned value for a radio button or checkbox.
placeholder=" " Placeholder It can provide a hint to the user of what can be entered in the field. It is in gray and disappears/reappears with input.
list=" " Data List It pairs an <input> with a <datalist> [id]. As the user types a list is updated to show options that best match the datalist options.
required Required It is a Boolean attribute indicating the user must specify a value for the form to be submitted. The user would see a message. 
<label   >
 for=" " For It declares the [id] of the paired <input> or <select> for that <label>. It's not needed if the <label> is wrapped around the <input>. 
<select   >
 name=" " Name Attribute has the same purpose and description as for the <input> element {above}.
id=" " ID Attribute has the same purpose and description as for the <input> element {above}. 
<option   >
 value=" " Value It sets an assigned value for a drop-down list or data list. {This may be different than the visible caption} 
<textarea   >
 name=" " Name Attribute has the same purpose and description as for the <input> element {above}.
id=" " ID Attribute has the same purpose and description as for the <input> element {above}.
rows=" " Rows It determines the height of the element.
cols=" " Columns It determines the width of the element.

Types of Input
Element & Attribute Value Name Description 
<input type=" ">
 "text" Text It renders a text field that users can type into. This is the default.
Constraint validations minlength=" " and maxlength=" " can message the user and prevent form submission.
"number" Number It allows the user to enter only numbers and a few special characters. It has built-in number validation.
Constraint validations min=" " max=" " step=" " can message the user and prevent form submission.
"date" Date It let the user enter a date, either with a textbox that validates the input or a special date picker interface.
Constraint validations min=" " max=" " step=" " can message the user and prevent form submission.
"password" Password It renders a single row input field and censors the sensitive text by displaying an asterisk (*) or a dot (•) instead.
When the form is submitted, the actual value of the text is sent.
"checkbox" Checkbox It renders a single checkbox item. A group of related checkboxes should all use the same [name] attribute.
If [checked] is added to the <input> tag it will default to being checked initially.
"radio" Radio Button It renders a single radio button. Related radio groups use the same [name] attribute but only one from a group can be chosen.
If [checked] is added to the <input> tag it will default to being checked initially. {This creates a required response}
"submit" Submit Button It is rendered as a button. When the click event occurs, the user agent attempts to submit the form to the server.
It can use value=" " for the button caption or it will use the user agent's default. {probably "Submit"}
"range" Range Slider It renders a visual widget to adjust between a minimum and a maximum value. The values are not displayed.
